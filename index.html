Project Idea

Overview

My project idea is a mobile application which I will call “JukeBox”. The application will essentially a combination of the music application Spotify and popular dating platform Tinder. The application presents the user with new songs they haven’t heard before based on their favourite genres and artists, which they are able to either swipe right to show they “like” the song, which automatically saves the name of the song and adds it to a pre-determined playlist, or they can swipe left to “dislike” the song. After every swipe, a new song will appear on-screen for the user to listen to until the application is closed.

Motivation

My reasoning for wanting to make this application is that I, as with most people, find it difficult to find new music / new artists to listen to. Majority of the music I listen to comes from already popular artists or are recommended to me by peers. I am not alone, it is estimated that over 50% of people listen to some form of music daily, most of which potentially run into the same issue where they’re just listening to the same music for hours on end, with no good way to find new music. An application where the user can find new music with just a few swipes either left or right would be super convenient and time efficient.

Description

The user would download the application just like any other application, which would be through either the Google PlayStore or the App Store. The user will then be asked to create an account so the song recommendations can be saved for that specific user. During the account creation process, the user will be asked a series of questions based around their music tastes and what artists they already currently listen to. Once set up, the application will give the user an option whenever or not they need a video tutorial on how to use the application. If declined, the user can get straight to using the app. If accepted, an embedded pre-recorded short video will play giving the user a basic outline on how to use the application. The user will also be given the option to upload a photo for their account profile.

Once set up, the app will start displaying music on-screen. It will show the album cover of the song and a playback bar. The user will be able to play, pause, rewind, and fast forward the song that is currently being displayed. Once the user feels they have decided on whenever or not they like the song, they can either swipe right to add the song to a “liked music” list or they can swipe left to dislike the song and move onto the next song. No matter if the user swipes left or right, there will be another new song for the user to listen to.

As stated, all songs that the user swipes right on is added to a “liked music” list, which the user can access at anytime whilst the application is open. The user would be able to re-listen to songs on their liked playlist and remove songs at any time. The user can also share songs on their liked music via a share button, which would give them the option to either share via text message, email, or copy the URL to send on any other messaging platform they wish.

Another feature of the application is to be able to search for profiles of other users of the application. The user will be able to type a username into a search bar and, if the username is typed correctly, will direct the user to the profile which they search for. On the profile page, it will display the profile photo and the music which that user liked. However, to ensure that the user has the right to privacy, there will be an option to be able to hide liked music on their public profile so other users cannot see it.

At any time, the user can access a settings menu and change their genre and artist preferences. This feature is implemented because the music tastes of people can drastically change overtime, and it is important to ensure that the application can help meet the new demands of the end user. In addition, to comply with the laws and regulations, there will be an option to delete the account of the user and an option to download all the data that the application collects on behalf of the user.

Tools and Technologies

Since this is a software application, the only physical hardware needed is a mobile device (preferably android) and a computer to write the programming on. The mobile device will be used to test the usability of the application as well as ensure that GUI is user-friendly and is being displayed properly. In terms of software, this will primarily be a web-based application. This means it will need HTML, CSS, and Java. It will also need a backend to be able to store the information, which means it will need a database manager like SQL. The information that will be stored is the login details of the users and their liked music.

Skills Required

The skills that will be required for my project idea will be Java Programming, Database Programming, HTML5 / CSS Web Programming, and API. A lot of skills required can be learned (relatively) easily through the internet. However, using an API might be challenging but, by using Youtube’s API, there should be plenty of documentation and video tutorials.

Outcome

If the project is successful, the outcome will be that people who consume music will be able to more easily find new songs and artists based on music they already like. This will help eliminate the frustration that comes with trying to find new music to listen to. Indirectly, this will help up and coming musical talent be able to reach a broader audience and help them gain more fans and listeners, increasing their popularity.
